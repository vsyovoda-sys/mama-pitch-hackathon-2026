## 0. TL;DR（一段话讲清是什么）

**妈宝（Mom-Bot）**是一款为"妈妈—女儿"二人组设计的 PWA（可装到主屏的网页 App），形态是一个**三方群聊**：成员 = 妈妈 + 女儿 + 女儿的 AI 助手。妈妈遇到说不清的手机问题时，在群里发语音/截图/录屏，AI 默认接管回答，遇到金钱 / 授权 / 多步骤 / 情绪高风险场景主动 @ 女儿介入；女儿一直在群里，看到 AI 处理有风险可以随时接管。**核心主张：AI 不替代母女关系，AI 站在女儿这一侧，把妈妈的"说不清"翻译成"说得清"。**

- **Slogan**：让妈妈在三方群聊里开口，AI 先接住，女儿在旁守着
- **产品名**：妈宝　**队名**：妈宝之家
- **赛事**：2026/05/09–05/10 北京母亲节黑客松（She Code Lab / Motherboard 生·无界）
- **路演时长**：8 分钟（含设备调试），demo day 当天 18:00 路演 Top12，20:00 出 Top6

> **写作/讨论纪律**：所有产品叙事都用"清泉/托托"代称，避免把妈妈写成"老人/小白"，避免把女儿写成"客服"。

## 2. 产品定位（必须守住的边界）

### 2.1 是什么
- 跨越代际和距离的**母女技术照护桥梁**。
- 一个妈妈、女儿、AI 平等存在的**三方群聊**。
- 隐私叙事：不收手机号 / 微信 / 邮箱；不走账号体系；Supabase 存储 + HTTPS；不打 E2EE 旗号，实话实说。

### 2.2 不是什么（高频被误判，需主动澄清）
- ❌ 不是适老化教程（不是"教妈妈用手机"）
- ❌ 不是替妈妈操作手机（不是远程接管）
- ❌ 不是某具体场景工具（不是地图/支付/购物工具）
- ❌ 不是 1v1 客服关系
- ❌ 不是泛 AI 陪伴 / 母婴社区 / 公开 SaaS
- ❌ 不做温馨花活（封面、贴纸、月报等）
- ❌ 不再以"本子 / 卡片复利"作为产品核心叙事（AI 自然有上下文记忆，但不是主张）
- ❌ 不打 E2EE / 家庭密钥旗号
- ❌ 不做实时屏幕共享主方案
- ❌ 不把账号/支付/验证码作为现场 demo 主场景

### 2.3 关键约束
- 妈妈侧：愿意开口 + 不觉得打扰女儿 > "用什么形式发出来"。
- 女儿侧：能和妈妈一起拆解，而不是替妈妈处理掉。
- AI 角色：**站在求助方的对侧**，识别页面、整理上下文、调本地应用、引导录屏；不替代关系。

## 3. 核心闭环（最小可演路径）

```
清泉打开 App（桌面图标，群聊视图）
 → 在群里语音说"我手机不知道扣什么钱了"
 → AI 立刻接住：可能反问、可能语音引导妈妈调起系统录屏（iOS 控制中心 / Android 下拉）
 → 妈妈录完回到群里，发录屏 + 一句语音补充
 → AI 多模态结构化解读现场（识别 App / 弹窗 / 卡点）
 → 能直接答就直接答；遇到金钱 / 授权 / 多步骤 / 情绪风险 → 主动 @托托
 → 托托在群里回复（文字/语音/截图画箭头）
 → 妈妈端语音播报方案 → 妈妈跟做
 → 解决后 AI 在群里发鼓励 + 闭环一句"下次还有这种事直接喊我"
```

**关键交互形态变更（2026/05/08）**：从最早设想的"妈妈 vs AI"单聊，改为**"妈妈 + 女儿 + 女儿的 AI"三方群聊**。这一改让"AI 站在女儿一侧"从概念变成了可视化产品形态，也直接化解了"和豆包同质化"和"微信可替代"两条最常见的评委质疑。

## 4. 技术形态决策表（已锁）

| 决策项 | 结论 | 理由 |
|---|---|---|
| 应用形态 | **PWA**（添加到主屏） | 28h 内唯一现实"App 形态"路径；iOS/Android 通吃 |
| 录屏方式 | 系统自带录屏 + 语音引导 + 相册回传 | iOS 完全禁止第三方录系统屏，借系统能力最稳 |
| 录屏回传 | Android: Web Share Target API；iOS: App 内选相册 | 妈妈熟悉"分享"和"选照片" |
| 语音播报 | 浏览器 SpeechSynthesis + TTS 兜底 | 首次授权后续直接用 |
| 语音输入 | Web Speech API + Whisper 兜底 | iOS Safari 中文识别一般，后端转写补 |
| 数据存储 | Supabase（demo 期） | 隐私叙事走"不收手机号/微信/邮箱"，不打 E2EE 旗号 |
| AI 模型 | GPT-4o / Gemini Vision（多模态） | 看视频帧 + 听音频 + 输出结构化 JSON |
| 配对方式 | 二维码扫码 | 妈妈不打字 |

### 黑客松当天**不做**（pitch 也不讲）
- 账号/注册/绑定流程
- E2EE 端到端加密 / 家庭密钥（整个方向已废弃）
- AI 真实操控其他 App（系统权限不允许）
- 完整社区互助平台、陆生人兜底层（架构图露出即可；pitch 讲远期邀请制志愿者社区）
- live demo 上传录屏的 AI 实时回复（烧 token 易翻车，走预录视频）
- 多人/多家庭群聊（demo 只演 3 人群）

## 6. Demo 选材（卡点素材库）

**选材原则**：
1. 真实发生过（来自队员/观众妈妈，不要编）
2. 妈妈当下**说不清**：步骤多、边界模糊、单截图讲不清
3. **已有适老化产品覆盖不到**（不是"放大字体""加常用联系人"这种入口型）
4. **低隐私**：不涉及支付密码、验证码、人脸、身份证、真实订单
5. 越能体现"母女一起拆"越好（不要"女儿一秒就能替她解决"的事）

**首选候选**：
- 抖音/美团/拼多多里**莫名其妙的会员订阅** —— 按钮藏几层、退订入口隐蔽、说不清是哪个 App 扣的
- 各类 App 的**弹窗扣费 / 权限索取陷阱**
- 短信里的快递通知（如丰巢/分场）→ 妈妈分不清真假 / 不知道操作流程

**绝对避开**：账号找回、支付/扣款、医院挂号、保险/理财、人脸认证。

**当前主推 demo 故事线（写进视频脚本）**：
妈妈在群里发丰巢取件短信截图 → AI 识别"分场关键字，疑似抖音订单退货" → 给步骤 → 妈妈说"找不到那个菜单" → AI 引导系统录屏 → 看到卡点继续指导 → 解决 → 妈妈反问"快递员什么时候上门、放哪儿"（涉及生活习惯，AI 答不了）→ AI 主动 @托托 → 托托语音回复确认时间和接驳点 → 解决 → AI 沉淀卡片备忘下次复用。

## 8. 调研依据（市场判断）

### 8.1 规模
- 2025 年中国 60+ 网民 **1.61 亿**，普及率 >52%
- 北京互联网法院：2018-2025 涉老网络消费纠纷案件 1009 件，**年均增长 23.5%**
- 工信部已推动 2577 家网站/App 完成适老化改造（2024）

### 8.2 已被覆盖 vs 真正空白
| 类别 | 代表 | 没覆盖什么 |
|---|---|---|
| 远程屏幕共享 | FaceTime / 微信视频 | 必须双方同时在线；妈妈仍要描述"我点了什么" |
| 远程控制 | 向日葵 / TeamViewer | 安装授权门槛高；侵入感强；妈妈不愿交手机 |
| 适老化模式 | 各家"长辈版" | 只解决"打开 App"那一步；不解决弹窗/订阅/扣费链路 |
| 家庭账号 | Apple 家人共享 / Family Link | 没有专门为"妈妈正卡住"设计的协作通道 |
| 通用 AI 助手 | 豆包 / Kimi / ChatGPT | 妈妈本身就**说不清**；AI 不在妈妈的现场 |

### 8.3 三个真正空白点（我们要打的靶）
1. 跨平台 + 跨 App 的**分步引导**（不是"看"，是"高亮+下一步+判断对错"）
2. **复杂操作链路解析**（"我也不知道我啥时候订的"这种链路长、入口模糊的问题）
3. **可控的代操作信任设计**（远控是"全开"、屏幕共享是"全看不动"，缺中间档）

### 8.4 中老年人手机使用四类系统性后果（pitch 背景板）
1. 基础操作不会（已被适老化部分覆盖）
2. **弹窗/诱导下载/流氓软件 → 手机变广告机**（最普遍；上海消保委测 600 款 App，58% 含广告，其中 69.7% 没"关闭键"）
3. 短视频/直播/私域 → 精神和钱包同时被收割（行为成瘾、抑郁躯体化）
4. AI 时代的换脸/拟声诈骗（弹窗→隐私权限被打开→数据进黑灰产→成为诈骗弹药库）

## 10. "本子"的复利逻辑（最关键的差异化叙事）

```
第 1 次：妈妈问 → AI 全包回答 → 沉淀卡片
第 N 次：妈妈问相似问题 → AI 先翻本子 → 命中 → 直接语音播报
        → 妈妈跟做 → 完成
```

**意义**：
- **女儿不是无止境的客服** —— 这是一个会"长大"的工具
- **学习成长故事线天然落地** —— 群聊有时间轴：第 1 次 AI 全包 → 第 N 次 AI 提示 → 妈妈自己完成 → AI 群里发花 → 女儿点赞
- **三层求助路由可视化** —— AI @女儿 / @社区好友 / @陌生人兜底，全部用群里 @ 的方式呈现，评委一眼看懂架构
