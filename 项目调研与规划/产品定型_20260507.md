> "三方群聊 + AI 智能分流 + 邀请制志愿者社区"，本子与 E2EE 整章已废弃。

## 1. 一句话

**妈宝是一个由"妈妈 + 女儿 + 女儿的 AI 助手"组成的三方群聊。妈妈遇到说不清的智能手机问题时随时在群里开口，AI 默认接管回应；遇到多步骤、前后矛盾、关乎安全的求助，AI 自动拉女儿入场；女儿一直在群里，看到风险也能直接接管。**

## 2. 妈妈端体验流（清泉视角）

```
1. 清泉打开"妈宝"群聊（桌面图标 / App）
2. 在群里发语音 / 截图 / 录屏："我手机不知道扣什么钱了"
3. AI 立即接住——可能直接答、可能反问澄清、可能引导她录屏
4. AI 调用本地应用 / 引导系统录屏 → 把"说不清"转成"看得见"
5. 如果是简单问题：AI 直接给到妈妈能听懂的语音方案
6. 如果是复杂 / 风险 / 需要女儿决策的问题：AI 在群里 @ 托托
7. 托托回复（文字 / 语音 / 在录屏截图上画箭头）→ AI 转成妈妈听得懂的语音播报
8. 解决 → 群聊里 AI 发一句"成啦，下次再有这种事直接喊我"
```

## 3. 女儿端体验流（托托视角）

```
1. 平时不在群里发言，AI 自动应对妈妈的大多数提问
2. 收到 AI @ 通知（PWA 推送 / 兜底邮件 / 微信兜底）才介入
3. 打开群聊看到：妈妈的求助 + AI 已经做的事 + AI 为什么需要她
4. 编辑回复：文字 / 语音 / 在录屏截图上画箭头
5. 看到 AI 处理过程中有安全风险（比如要付款 / 要授权敏感权限），可以随时直接喊停
```

## 4. AI 智能分流的判断逻辑

AI 默认接住，下列情况主动 @ 女儿：

- **金钱相关**：付款、贷款、扣费、退款、订阅等关键词
- **隐私授权**：要求开启位置 / 通讯录 / 摄像头等敏感权限
- **多步骤连环操作**：超过 3 步且前一步结果会影响后一步
- **妈妈表达出焦虑 / 委屈情绪**：声音颤抖、反复重复同一句、"我是不是被骗了"
- **AI 自己不确定**：识别置信度低于阈值

## 5. 下一步：邀请制志愿者社区（黑客松后规划）

- **应用场景**：女儿不在线 / 不方便 / 短期出差时
- **接入方式**：女儿提前邀请 1-2 位"她信任的朋友"加入这位妈妈的"备份名单"
- **激活机制**：AI 判断需要人介入但女儿超时未响应 → 在备份名单里轮询 → 接通后 AI 同步上下文
- **兜底**：如果整条名单都失联，AI 给妈妈一个客服电话，并且在群里留言告诉女儿"我已经帮她联系了客服"

## 6. 技术形态选择 · 一表说明

| 决策 | 结论 | 理由 |
|---|---|---|
| 应用形态 | PWA / App | 28h 内 PWA 是唯一现实路径；上线后可包成 App |
| 群聊容器 | 自建（不寄生在微信） | 三方群聊里 AI 是平等成员，微信里实现不了 |
| 录屏方式 | 系统自带录屏 + AI 语音引导 | iOS 禁止第三方录系统屏；借系统能力 |
| 录屏回传 | Android Web Share Target；iOS 选相册 | 妈妈熟悉"分享"按钮 |
| 语音 | SpeechSynthesis + Whisper 兜底 | 第一次授权后续直用 |
| AI 模型 | GPT-4o / Gemini Vision（多模态） | 看视频帧 + 听音频 + 输出结构化 |
| 数据存储 | Supabase | demo 期；隐私叙事在产品页正常表述（用 HTTPS / 不卖给第三方），不打"E2EE"旗号 |

## 7. 关卡与应对

| 关卡 | 现实 | 应对 |
|---|---|---|
| PWA 不能直接调起系统录屏 | iOS/Android 都不行 | AI 语音引导妈妈手动开 + 相册回传 |
| iOS PWA 后台推送 | iOS 16.4+ 才支持 | 首次引导加桌面 + 通知；fallback 邮件 / 短链 |
| 中文语音识别质量 | iOS Safari 一般 | 录音上传 → Whisper 兜底 |
| 录屏视频很大 | 几十 MB | 上传压缩 + 分片；后端 AI 抽帧 |
| AI 误判要不要叫女儿 | 可能 | 默认偏保守，宁可多叫；女儿可以一键说"这种以后不用叫我" |
| 路演现场网络不稳 | 高风险 | 录播视频兜底 + 现场只演关键 30 秒 |
