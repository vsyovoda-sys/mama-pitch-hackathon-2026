# 第一轮调研结果：替代品与脚手架

基准：产品认知对齐.md
状态：已完成第一轮桌面调研；保留作为"为什么不能只做远控"的证伪材料。

## 1. 当前最重要结论

实时屏幕共享不是唯一入口，也不一定是最适合妈妈的入口。

更接近当前核心目标的方向是：妈妈一键生成“求助包”，女儿通过链接异步查看上下文，再决定是否文字回复、电话沟通、转交可信朋友，或升级为实时协助。

## 1.1 方案成立性边界

少数派《爸妈的手机有问题，你可以怎么帮他们？》说明，苹果、华为、小米、OPPO/vivo/魅族、TeamViewer、AnyDesk、向日葵、腾讯会议等，已经覆盖了“双方同时在线，远程看屏或远程控制”的大量场景。

因此，如果本项目只做“女儿远程看妈妈手机屏幕并指导操作”，方案不成立，已有产品已经做得更完整。

本项目仍可能成立的边界是：

- 妈妈不想打扰女儿，或女儿当下不一定能接。
- 妈妈不知道怎么描述问题，需要产品帮助她复现并打包上下文。
- 求助内容可以先异步发给女儿，女儿有空再看；必要时再升级为通话或转交可信朋友。
- AI 不做远程控制，而是帮助女儿理解截图/录屏、识别风险、整理下一步。

结论：项目不能再以“远程协助”作为核心创新点，核心应转为“低负担异步求助包 + 女儿侧 AI 辅助判断”。

## 2. 已有替代品覆盖了什么

### 2.0 Bug 上报工具的关键启发

代表：Jam、Bird Eats Bug、Userback、BetterBugs、Loom。

这些工具的共识是：用户很难用文字准确描述问题，最有效的方式是录下复现过程，并自动打包上下文。Jam、Bird Eats Bug、Userback 等会把屏幕录制、操作步骤、日志、环境信息合在一个报告里，减少支持方反复追问。

迁移到本项目：妈妈不需要选择场景类型，也不需要先解释清楚；产品通过语音提示她“把刚刚的问题再演示一遍”，自动生成求助包。

### 2.1 远程协助/远程控制

代表：TeamViewer QuickSupport、ToDesk、向日葵、AnyDesk、AirDroid/AirMirror、华为亲情关怀、FaceTime 屏幕共享。

它们解决的是“帮忙的人看不到屏幕”的问题，常见机制包括连接码、设备 ID、授权确认、实时语音、屏幕共享、部分远程控制或涂鸦标注。

不足：安装和授权步骤偏重，往往要求双方同时在线；移动端权限复杂；iOS/Android 能力不一致；对妈妈来说，求助前的准备成本仍然高。

### 2.2 异步录屏/链接工具

代表：Jam Recording Links、Loom。

它们解决的是“用户说不清问题，支持方需要看到现场”的问题。Jam 的机制尤其接近：发一个链接，用户点击录制，系统把屏幕内容和上下文打包回传；Loom 则强调录屏、分享链接、转写和 AI 摘要。

不足：主要面向团队协作、软件调试或客户支持，不是家庭技术求助；妈妈端仍需要被改造成更低门槛的一键入口。

### 2.3 人类视觉辅助

代表：Be My Eyes、Aira、Seeing AI。

它们证明了“人 + AI + 视觉上下文”可以成立。Be My Eyes 通过实时视频把需要帮助的人连接到志愿者或企业，并提供 AI 工具；它的启发是低门槛求助、真实人介入、只分享必要画面。

不足：场景是视觉障碍辅助，不是手机 App 技术问题；交互更偏实时，不解决“我不想现在打扰女儿”的异步压力。

## 3. 技术脚手架判断

### 3.1 Web 实时屏幕共享风险高

MDN 对 `getDisplayMedia()` 的兼容性显示：桌面浏览器支持较好，但 Chrome Android、Safari iOS、移动 WebView 不支持。也就是说，如果黑客松用纯网页实现“妈妈手机实时共享屏幕”，风险很高。

### 3.2 一天内更稳的 Demo 路线

优先做“异步求助包”而不是完整远控：

1. 妈妈端：上传截图/录屏，补一句语音或文字说明。
2. 系统：生成一个求助链接。
3. 女儿端：打开链接，看上下文。
4. AI：识别截图/录屏信息，整理问题、可能路径、风险提示。
5. 女儿：选择回复方式，给妈妈一段更容易执行的建议。

这条路线不需要攻克移动端实时屏幕共享，但 MVP 必须真实可用：妈妈至少能上传/拍摄截图或录屏，补充语音，生成链接；女儿能打开链接查看，并获得 AI 辅助摘要。

## 4. 对产品方向的修正

- MVP 不要先绑定“实时通话 + 屏幕共享”。
- 核心对象不是屏幕，而是“妈妈说不清的问题上下文”。
- 求助包是当前最大公约数，但妈妈端必须表现为一种单一动作：复现问题、录下来、补充语音、发给女儿。
- 源头不明型和账号安全型更复杂，适合放进 Pitch 作为后续验证场景。
- 转交可信朋友是有潜力的扩展，但必须有权限、有效期、隐私提示。

## 5. 下一轮调研要做什么

1. 做 8-10 张场景卡，验证问题分类是否够用。
2. 挑 3 个代表场景测试 AI：妈妈原话、截图/录屏、女儿整理后三种输入分别能得到什么结果。
3. 查可直接搭 Demo 的工具链：截图/视频上传、链接分享、AI 视觉理解、语音转文字、页面原型。
4. 对比“异步求助包”和“实时屏幕共享”在妈妈负担、女儿负担、Demo 难度上的差异。
