# 产品认知对齐（已根据 0510 收敛重写）

用途：给队友和 Agent 快速对齐当前产品定型。其他历史方案、调研论证另文承接。

## 一句话

**妈宝是一个由"妈妈 + 女儿 + 女儿的 AI 助手"组成的三方群聊。妈妈遇到说不清的智能手机问题时随时在群里开口，AI 默认接管回应；遇到多步骤、前后矛盾、关乎安全（金钱 / 授权）的求助，AI 自动拉女儿入场；女儿一直在群里，看到风险也能直接接管。**

## 主角代称

- **托托**：女儿。会一点 AI、是这个 App 的部署/管理者。
- **清泉**：妈妈。隐含用户画像 = 会用智能机但说不清问题、信任女儿、不愿打扰女儿。
- **群聊里的 AI**：女儿"训练 / 配置过"的助手，默认替女儿在线。

## 三方群聊的关系结构

- **妈妈 → AI**：日常 90% 求助，AI 直接接住（截图识别 / 录屏引导 / 调本地应用）
- **AI → 女儿**：剩下 10% 涉及安全 / 决策 / 情绪 → AI @ 女儿入场
- **女儿 → 群聊**：随时可介入，看到 AI 处理有风险时直接喊停或接管

## 产品形态

- **PWA / App**（添加到主屏，对清泉 = 一个 App 图标）
- 全语音引导，清泉不需要看文字识别按钮
- 录屏调用系统能力（iOS 控制中心 / Android 下拉），不自研
- 数据存储用 Supabase；隐私叙事走"不卖数据 / 不收手机号微信邮箱"，不打 E2EE 旗号
- 不做公开 SaaS、不做泛 UGC 社区
- 远期会有"邀请制志愿者社区"——女儿不在线时由女儿邀请的可信朋友兜底

## 核心闭环（必须保的最小可演路径）

```
清泉在群里语音："我手机不知道扣什么钱了"
 → AI 立刻接住、反问 / 引导她录屏
 → 清泉用系统录屏，回到群里发出视频
 → AI 多模态结构化解读 → 在群里给出语音方案
 → 如果是付款 / 授权 / 复杂情况 → AI 在群里 @ 托托
 → 托托介入回复 → AI 转译成清泉听得懂的语音播报
 → 解决 → 群聊里 AI 发一句"成啦"
```

## 不是什么

- 不是适老化教程
- 不是 AI 替清泉解决问题（AI 是把"说不清"变"说得清"，不是替代沟通）
- 不是远程接管手机
- 不是某具体场景工具（不是地图工具、不是支付工具）
- 不是 1v1 客服关系
- 不是泛 UGC 母婴社区
- 不做温馨花里胡哨的 workspace
- 不再做"本子 / 卡片复利"作为核心叙事——AI 当然有上下文记忆，但不是产品主张
- 不打"端到端加密 / 家庭密钥"旗号——黑客松不实做，pitch 也不讲

## 关键约束

- 清泉侧：愿意开口 + 不觉得打扰托托，比"用什么形式发出来"更重要
- 托托侧：能和清泉一起拆解，而不是替清泉处理掉
- AI 角色：站在女儿一侧，帮理解现场（识别页面 / 整理上下文 / 调本地应用 / 引导录屏），并在风险时主动叫女儿；不替代关系
- 不做实时屏幕共享主方案
- 不做泛 AI 陪伴
- 不把账号 / 支付 / 验证码作为现场主 Demo
