# 赛事评审突围策略 v2（基于产品定型版）

基准：No.C004「Motherboard 生·无界」黑客松选手手册、产品认知对齐.md、产品定型_20260507.md

---

## 1. 手册给出的真实评审信号

这不是只比技术完成度的黑客松。手册反复强调的是：女性视角、真实需求、被忽视的经验、快速原型和 Demo 展示。

评审维度只有三个：

- **共鸣性**：是否触及真实女性经验，让被忽视的痛点、情感、劳动被看见。
- **创新性**：是否在问题定义、技术用法、交互形式或跨边界连接上出人意料。
- **实用性**：是否真正降低门槛，让用户完成以前独自做不了的事。

奖项权重提示：

- **生·命力奖**：共鸣性最高，适合情感强、让人"被看见"的项目。
- **无·界奖**：创新性最高，适合打破代际、身份、技术边界的项目。
- **成·长奖**：实用性最高，适合让女性获得能力提升的工具。

我们的项目天然最适合主打"连接"，同时争取生·命力奖或无·界奖。如果现场 Demo 足够真实、本子的复利逻辑讲清楚，也可以兼顾成·长奖。

## 2. 项目要让评委记住的一句话

> 妈妈不是不会，她是说不清自己卡在哪里；女儿不是不帮，她是没办法和妈妈一起站在那个屏幕前。

这个项目解决的不是"教妈妈用手机"，也不是"替女儿远程控制妈妈手机"，而是让妈妈愿意把任何一个说不清的卡点拿出来，和女儿一起把它拆开、看懂、解决——问题被解决，关系也被重新连接一次。

**AI 在这里不替代关系，AI 站在女儿这一侧，把妈妈的"说不清"翻译成"说得清"。**

---

## 3. 项目定位

主叙事：**跨越代际和距离的母女技术照护桥梁**。

具象表达：

- 妈妈端是一个全语音引导的网页 App（妈妈装到桌面就是一个图标），她不用读文字、不用辨认按钮。
- 女儿端是同一个 App 的另一个视角，看见妈妈的现场（录屏 + 语音 + AI 结构化解读）后回复。
- 解决一次就在"我们的本子"里多一张卡片，下次相似问题 AI 先翻本子，能答就直接答——这是**让女儿被一点点解放**的复利逻辑。
- 端到端加密，不收手机号 / 微信 / 邮箱——**只属于妈妈和女儿俩**。

副叙事可选：双向通道——妈妈→女儿（手机）+ 女儿→妈妈（生活），关系不只单向。

## 4. 三个维度怎么打高分

### 共鸣性

要讲一个非常具体的生活瞬间：**妈妈遇到手机问题时常去手机店，因为那里的人能直接看到问题，也不会让她觉得自己在打扰女儿。**

现场表达要避免把妈妈描述成"落后"或"老年人"。更准确的说法是：她不是没有学习能力，而是数字产品把问题藏在层层页面、弹窗、权限、订单和路线里，她很难把现场翻译成一句聊天消息。

路演里最好出现 1-2 句妈妈原话，例如：

- "我不知道怎么说，反正它就卡在这了。"
- "你忙的话我就不问你了，我去手机店看看。"
- "我怕我点错了，扣钱怎么办。"

**附加共鸣点**：女儿端那侧"长期承担但不被命名的技术照护劳动"也要被点名一次——这是隐藏的女性经验。

### 创新性

不要把 AI 做成"妈妈的聊天机器人"，也不要做成"替妈妈解决问题的客服"。这类方案评委一秒识别为普通。

我们的创新点有三层，路演要至少打到两层：

1. **AI 不替代亲密关系**——AI 站在女儿这一侧，帮女儿更快看见妈妈的现场。
2. **本子的复利逻辑**——AI 先翻本子能答就直接答，意味着女儿不是无止境的客服。这是一个会"长大"的工具。
3. **端到端加密 + 无账号**——在 2026 年，做一个不收用户信息、服务器看不见任何数据的 App，本身就是反主流。"我们没有用户表，只有家庭密钥。"

### 实用性

必须证明它不是只有故事的概念。

第一版真实可用链路必须跑通：**妈妈打开 App → 语音引导 → 调起系统录屏 → AI 解读 → 女儿回复 → 妈妈端语音播报 → 自动留卡片**。这一条要在路演现场用真机演完。

不要承诺技术上跑不通的事。例如"App 自动录其他 App 的屏幕"——iOS 系统层面禁止，路演时要诚实承认这一点，并展示我们的解法："我们不绕系统，我们用语音教妈妈调起系统录屏。"

## 5. 用什么具体卡点做 Demo

产品**定位上不绑场景**——它服务的是"妈妈手机里说不清的卡点"这一类问题，而不是某一个具体功能。但路演必须有一个真实卡点把故事打穿。

选材原则：

- 必须**真实发生过**，最好来自队员或现场观众的妈妈，不要编。
- 必须是妈妈当下**说不清**的：前置步骤多、问题边界模糊、单张截图或一句话讲不清。
- 必须**已有工具明显覆盖不到**：不要选"放大字体"、"加常用联系人"、"打开地图"这类已被适老化产品做掉的入口型问题。
- 必须**低隐私**：不涉及支付密码、验证码、身份证、人脸认证、真实订单和真实账号。
- 越能体现"妈妈和女儿一起把它拆开"越好，不要选"女儿一秒就能替她解决"的事。

**首选候选**（基于真实素材）：

- 抖音 / 美团 / 拼多多里**莫名其妙的会员订阅**——按钮藏几层、退订入口隐蔽、说不清是哪个 App 扣的。
- 各类 App 的**弹窗扣费 / 权限索取陷阱**。

赛前准备 2-3 个真实卡点作为素材库，路演时挑一个最贴的用。

绝对避开：账号找回、支付/扣款、医院挂号、保险/理财、人脸认证。

## 6. 28 小时 MVP 必须达到什么

形态已锁（产品定型 § 6）：PWA / 全语音 / 系统录屏 / E2EE / 家庭密钥。

MVP 必须真实跑通：

- **妈妈端**：动作越少越好。语音引导从打开 App → 描述问题 → 教调起系统录屏 → 接收录屏 → 推送方案。妈妈不需要读文字、不需要辨认按钮、不需要做任何配置。
- **女儿端**：能快速看见妈妈的录屏 + 语音 + AI 结构化解读，回复一段文字 / 语音。
- **本子**：解决后自动生成一张卡片；AI 在新求助进来前先翻本子。
- **配对**：女儿端生成二维码 → 妈妈扫码完成家庭密钥配对，妈妈不打字。
- **AI 失败要有明确降级**：识别失败时直接走"原始录屏 + 妈妈语音"给女儿，不把规则模板伪装成 AI 输出。

不做：账号体系、好友网络、社区、SaaS 注册、月报、封面定制、贴纸等温馨花活。

详细模块拆解和"砍优先级"见 产品定型_20260507.md § 8。

## 7. 8 分钟路演结构

总时长按手册 8 分钟来设计。具体分配：

```
0:00-1:00  开场钩子 + 真实故事
   "妈妈不是不会用手机，她只是说不清自己卡在哪。"
   讲妈妈去手机店那一幕——那里的人能看见她卡在哪，也不会让她觉得在打扰女儿。

1:00-1:30  双向通道理念
   主轴妈妈→女儿（手机），副轴女儿→妈妈（生活）。
   "技术问题被解决，关系也被重新连接一次。"

1:30-4:30  现场 Demo（核心 3 分钟，真机+投屏）
   走一次完整闭环：妈妈端打开 → 语音说卡点 → 引导录屏 → AI 解读 → 女儿端回复 → 妈妈端语音播报方案。
   选材用赛前准备的 2-3 个真实卡点里最贴的一个。

4:30-5:30  本子 & AI 先查本子的复利逻辑
   展示已经累积的几张卡片。
   "下次妈妈问相似问题，AI 先翻本子，能答就直接答——
    这意味着女儿不是无止境的客服。这是一个会长大的工具。"

5:30-6:30  隐私 = 端到端加密
   "我们没有用户表、没有手机号、没有微信登录。
    我们的服务器看不见妈妈说了什么，也看不见女儿回了什么——
    只有妈妈和女儿手里的钥匙能打开。"
   兑现"只属于你们俩"。

6:30-7:30  团队 + 28 小时怎么做的 + 未来形态
   PWA + 系统录屏 + Web Crypto。
   未来形态一句带过：iOS Shortcut 自动录屏 / Android 原生 App。

7:30-8:00  收尾一句
   "母亲节我们想送的不是手机课，是一本会变厚的本子。"
```

**为什么不是现有方案**（嵌在 Demo 段后或评委问答里）：

- **远程屏幕共享**要求双方同时在线，且不解决"妈妈不想打扰女儿"。
- **聊天 AI** 要求妈妈说清楚问题。
- **微信截图**缺上下文，每次重新描述。
- **适老化模式**只解决入口，不解决"她已经在路上但下一步不会"。

## 8. 现场组队和执行策略

形态已定，组队策略变得简单：

**只有自己**：

- 砍掉女儿端在录屏截图上画箭头、卡片语义搜索、卡片蓝橙双向区分。
- 保留必做核心：PWA + 配对 + E2EE + 妈妈端语音循环 + 录屏回传 + AI 解读 + 女儿端文字回复 + 自动卡片 + 翻本子。
- Pitch 和评委问答准备到很稳。

**+ 1 个工程师**：

- 工程师吃技术全链：Supabase / E2EE / 多模态 AI 接入 / 录屏文件处理。
- 你做产品定义、卡点素材、UI 微调、路演结构和评委问答。
- 把"加分项"放回来：卡片蓝橙双向区分、女儿端画箭头。

**+ 设计师**：

- 妈妈端要极简，像一个她敢按下去的按钮。
- 女儿端要像和妈妈"一起看"，不要像工单系统。
- 本子卡片视觉要承载"会变厚"的复利感。

**+ 演讲人 / 产品**：

- 把 8 分钟稿件结构化、上台呈现。
- 分担评委问答压力。

## 9. 赛前可以准备什么

根据 Fresh Code 原则，核心代码、设计稿、原型图必须在 28 小时内现场创建。赛前可以准备调研、故事、技术选型、接口账号、数据结构、路演提纲和任务拆解，但**不要把可参评的核心 Demo 提前做完**。

建议赛前准备：

- **2-3 个真实妈妈卡点**（让自己妈妈或队友妈妈口述一段+截一张图，不要编造）。
- **接口账号准备好**：OpenAI（GPT-4o）/ Gemini Vision / Whisper / Supabase / Vercel，空白仓库建好，Key 准备好。
- **PWA 相关技术资料**：Web Speech API / Web Crypto API / Web Share Target API 的浏览器兼容性预案打印出来。
- **8 分钟路演提纲**先写好，留 Demo 选材的位置空着。
- **真机准备**：一台 Android 手机（演 Web Share Target）+ 一台 iPhone（演 PWA + iOS Shortcut 兜底）。
- **录播视频兜底**：现场万一网络不稳，能立刻切到提前录的 Demo 视频。

## 10. 不要做什么

- 不要把项目讲成"适老化教学工具"。
- 不要把妈妈讲成技术弱者或老年人。
- 不要做泛泛的 AI 陪伴或 AI 问答。
- 不要承诺"App 自动后台录其他 App 屏幕"——iOS 系统禁止，路演时坦诚承认并展示我们的解法。
- 不要把第一版做成社区平台 / 公开 SaaS。
- 不要选支付、验证码、身份证、人脸认证做现场主 Demo。
- 不要让 AI 直接对妈妈说话替代女儿；AI 站在女儿这一侧。
- **不要做温馨花活**（封面定制 / 贴纸 / 月报 / 烧本子按钮）—— 产品认知对齐已明确。

## 11. 最终判断

这个项目要突出重围，不能靠"功能多"，而要靠四个尖锐、能被立刻理解的切口的叠加：

1. **妈妈卡住的那一刻，母女终于能一起站在那个屏幕前。**（共鸣）
2. **AI 不替代关系，AI 是把"说不清"翻译成"说得清"。**（重新定义 AI 角色）
3. **本子让女儿被一点点解放，不是无止境的客服。**（复利逻辑）
4. **没有用户表，只有家庭密钥。**（隐私的反主流姿态）

黑客松现场最强的交付形态是：**一个真机能跑通的小 Demo，加一段非常准的母女关系叙事，加一个会让评委愣一下的隐私承诺**。

> 母亲节我们想送的不是手机课，是一本会变厚的本子。
